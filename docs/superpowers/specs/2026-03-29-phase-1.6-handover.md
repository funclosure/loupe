### 2. Markdown Editing UX — Heading Level Bug

**The problem:** When the user has a rendered `## Title` (H2) and tries to add more `#` characters to change the heading level, Milkdown's behavior breaks at H4+.

Example flow:
1. User types `## Title` → renders as H2
2. User places cursor before "Title" and types `#` → should become `### Title` (H3)
3. User types another `#` → should become `#### Title` (H4)
4. **Bug:** H4 is not supported (or falls through to body text) — the heading resets or renders incorrectly

**Expected behavior:** Either support H4-H6, or when the user tries to go beyond the max heading level, cycle back to H2 (or paragraph). The editor should never produce broken/unstyled output.

**Files to investigate:**
- `src/client/editor/milkdown-setup.ts` — Milkdown plugins, may need heading level configuration
- `src/client/index.css` — only H1-H3 have styles defined (lines 141-164), H4-H6 are unstyled
- Milkdown's commonmark preset — check if heading levels can be constrained
